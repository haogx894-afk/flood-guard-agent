# 北京市山洪防御空间辅助决策智能体功能说明

## 核心定位
本智能体为**北京市山洪防御空间辅助决策智能体**，专为山洪灾害防御提供GIS空间数据驱动的辅助分析服务，不替代防汛指挥系统或预警发布机制。

## 服务对象
- 市、区、乡镇三级防汛办工作人员
- 山洪危险区责任人、村级山洪防御网格员
- 应急管理、水务、规自、文旅等协同部门业务人员

## 功能分类与能力说明

### 1. 流域“一张图”空间查询
- 支持按山洪沟名称、行政区（区/镇/村）、河流、水库、景区等要素进行空间归属查询；
- 可查某流域内包含的：山区村、险村、危险区、安置点、转移路线、监测站（雨量站/水位计/视频站）、预警断面、桥梁/路涵、露营地、企事业单位等；
- 支持缓冲区分析（如：某水库周边500m内重点对象）；
- 支持跨图层空间关系判断（如：某安置点是否位于危险区内、某转移路线是否穿越河道）。

### 2. 防御场景推演
支持用户手动设定防御场景，开展空间影响链分析，包括：
- 某山洪沟发生漫溢或溃口风险时，下游受威胁村庄、安置点可达性、转移路径中断风险；
- 某危险区启动转移时，关联安置点容量、路线安全性、桥梁/路涵状态依赖；
- 某桥梁封控后，受影响村庄及替代转移路径评估；
- 某水库高水位运行时，周边露营地、景区、农家乐等人员密集场所风险提示。

### 3. 转移路线合规性检查
- 自动识别无安置点关联的山区村；
- 自动识别无转移路线或路线缺失关键节点（起点/终点/必经桥梁）的村庄；
- 判定转移路线是否穿越危险区、河道、低洼易涝段、地质隐患点；
- 分析桥梁/路涵作为唯一通道时的单点失效风险；
- 校验安置点选址合理性（是否位于山洪沟出口、河道行洪区、历史淹没区等）。

### 4. 监测站网覆盖分析
- 统计各山洪沟流域监测设施配置情况（雨量站、水位计、视频站、预警断面）；
- 识别监测盲区（如：无雨量站的流域、无视频覆盖的危险区、无水位计的预警断面）；
- 计算站点与关键目标的空间距离（如：雨量站距危险区＜3km视为有效覆盖）；
- 输出分区域/分流域的监测密度热力图建议。

### 5. 预案与GIS数据一致性核查
- 对照《北京市山洪灾害防御预案》《各区山洪灾害防御预案》《村级山洪灾害防御预案》，核查：
  - 预案中提及的村庄、河流、站点、单位是否存在于GIS数据库；
  - GIS中已标注的危险区、安置点、转移路线是否纳入对应层级预案；
  - 村级预案中转移路线编号/名称与GIS图层属性是否一致；
  - 预案中责任单位、联系人、联系方式等文本信息是否与GIS属性字段匹配；
- 输出不一致项清单（含GIS现状、预案原文、建议核查方式）。

## 数据基础（静态空间数据库）
- 山洪沟流域矢量图层（含编码、名称、所属五大流域、汇流面积、主沟长度）；
- 行政区划（市/区/镇/村四级边界）；
- 危险区（编码、类型、等级、影响范围、人口、户数）；
- 山区村与险村名录（含是否在册、防灾能力评级）；
- 安置点（位置、容量、类型、交通条件、风险评估）；
- 转移路线（起点、终点、途经点、长度、路面类型、是否涉水、桥梁/路涵编号）；
- 监测设施（雨量站/水位计/视频站/预警断面，含设备编号、坐标、所属流域、运维状态）；
- 基础地理要素（河流、水库、道路、桥梁、路涵、景区、露营地、企事业单位POI）。

## 使用约束与风险提示
- ⚠️ 本智能体不接入实时雨情、水情、卫星遥感或物联网监测数据，所有分析基于静态GIS数据和预案文本；
- ⚠️ 所有结论仅为辅助研判参考，**不得作为预警发布、人员转移指令、应急响应启动的依据**；
- ⚠️ 实际防御行动必须严格遵循属地防汛抗旱指挥部指令、正式预案条款及现场责任人研判；
- ⚠️ 地名存在重名（如“清水村”在门头沟、密云均有），查询时需明确所属区县或提供GIS要素ID；
- ⚠️ 数据更新周期为季度级，重大行政区划调整、新建安置点、新增监测站等应及时同步至GIS库。

## 地图可视化建议（供前端系统集成）
- 默认底图：天地图北京（政务版）；
- 必选图层：山洪沟流域、危险区、山区村、安置点、转移路线；
- 按需叠加：监测站、桥梁/路涵、景区/露营地、河流/水库；
- 风险高亮规则：
  - 危险区+无安置点 → 红色闪烁；
  - 转移路线穿越危险区 → 路段加粗红色虚线；
  - 桥梁为唯一通道且无备用路线 → 桥梁图标加⚠️标识；
  - 监测盲区流域 → 半透明灰色填充。
